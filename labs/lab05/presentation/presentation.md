# Цель работы

## Цель

- Получить навыки настройки VLAN
- Изучить VTP и trunk-порты
- Проверить передачу ICMP в сети

# Ход выполнения

## Построение топологии сети

- Создана корпоративная сеть в Packet Tracer
- Использованы коммутаторы, ПК и серверы
- Сеть имеет иерархическую структуру

![Топология сети](Screenshot_1.png){ width=78% }

## Настройка IP-адресации

- На ПК выбран режим **Static**
- Заданы IP-адрес, маска и шлюз
- Адреса распределены по подсетям VLAN

![Настройка IP на ПК](Screenshot_2.png){ width=78% }

## Примеры назначенных адресов

- **10.128.3.10/24** — сегмент dk
- **10.128.4.10/24** — сегмент departments
- **10.128.5.10/24** — сегмент adm
- **10.128.6.10/24** — сегмент other

![Статическая адресация](Screenshot_6.png){ width=78% }

## Настройка VTP-сервера

- **msk-donskaya-ngaforov-sw-1** переведён в режим server
- Заданы домен **donskaya** и пароль **cisco**
- Созданы VLAN и их имена

![VTP-сервер и VLAN](Screenshot_8.png){ width=82% }

## Созданные VLAN

- **VLAN 2** — management
- **VLAN 3** — servers
- **VLAN 101–104** — пользовательские сегменты
- База VLAN хранится на сервере VTP

![Создание VLAN](Screenshot_8.png){ width=82% }

## Настройка trunk-портов

- Межкоммутаторные линии переведены в **trunk**
- Trunk передаёт трафик нескольких VLAN
- Использованы интерфейсы **g0/1**, **g0/2**, **f0/24**

![Trunk-порты на коммутаторе](Screenshot_10.png){ width=82% }

## Настройка VTP-клиентов

- Остальные коммутаторы переведены в режим client
- Указаны тот же домен и пароль
- VLAN получены автоматически от сервера

![VTP-клиент](Screenshot_9.png){ width=82% }

## Назначение портов VLAN

- Пользовательские порты настроены как **access**
- Порты распределены по VLAN 101–104
- Серверные порты отнесены к **VLAN 3**

![Назначение VLAN на порты](Screenshot_12.png){ width=82% }

## Проверка связности в одном VLAN

- Выполнен **ping** между узлами одной VLAN
- Все пакеты доставлены успешно
- Потерь при передаче нет

![Успешный ping](Screenshot_13.png){ width=78% }

## Проверка между разными VLAN

- Выполнен **ping** между разными VLAN
- Получено сообщение **Request timed out**
- Без маршрутизации доступ отсутствует

![Недоступность между VLAN](Screenshot_14.png){ width=78% }

## Режим симуляции

- Изучено движение ICMP-пакета по сети
- Отслежен путь через коммутаторы
- Просмотрен список событий передачи

![Simulation Mode](Screenshot_15.png){ width=82% }

## Структура передаваемого пакета

- Рассмотрен кадр **Ethernet II**
- Изучен заголовок **IPv4**
- Проанализировано сообщение **ICMP**

![Структура ICMP-пакета](Screenshot_16.png){ width=82% }

## Ethernet, IP и ICMP

- Ethernet содержит MAC-адреса источника и получателя
- IP задаёт адресацию и TTL
- ICMP используется для диагностики сети

![PDU Information](Screenshot_16.png){ width=82% }
