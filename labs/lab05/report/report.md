Для анализа структуры передаваемого пакета было открыто окно **PDU Information**, где отображается информация о протоколах, участвующих в передаче данных. На вкладке **Inbound PDU Details** показана структура кадра и вложенных протоколов.

Передаваемый пакет состоит из нескольких уровней модели OSI.

**1) Ethernet II (канальный уровень)**  
Кадр Ethernet содержит служебную информацию, необходимую для передачи данных между сетевыми устройствами:

- Preamble — последовательность битов для синхронизации передачи;
- SFD (Start Frame Delimiter) — признак начала кадра;
- Destination MAC — MAC-адрес получателя;
- Source MAC — MAC-адрес отправителя;
- Type — тип протокола верхнего уровня (**0x0800**, что соответствует IPv4);
- Data — полезная нагрузка кадра;
- FCS — контрольная сумма кадра для проверки целостности данных.

**2) IP (сетевой уровень)**  
IP-заголовок содержит параметры маршрутизации пакета:

- Version — версия протокола (IPv4);
- IHL — длина заголовка;
- DSCP — поле приоритета обслуживания;
- Total Length — общий размер пакета;
- Identification — идентификатор пакета;
- Flags и Fragment Offset — параметры фрагментации;
- TTL — время жизни пакета;
- Protocol — тип протокола верхнего уровня (**0x01 — ICMP**);
- Source IP — IP-адрес отправителя;
- Destination IP — IP-адрес получателя;
- Header Checksum — контрольная сумма заголовка.

**3) ICMP (сетевой диагностический протокол)**  
Протокол ICMP используется для диагностики сети и передачи служебных сообщений:

- Type — тип сообщения (например **0x08 — Echo Request**, **0x00 — Echo Reply**);
- Code — дополнительный код сообщения;
- Checksum — контрольная сумма;
- Identifier — идентификатор запроса;
- Sequence Number — номер последовательности пакета;
- Data — полезные данные сообщения.

## Контрольные вопросы

### 1. Какая команда используется для просмотра списка VLAN на сетевом устройстве?

Для просмотра списка VLAN на коммутаторе используется команда:

show vlan brief

Эта команда выводит таблицу всех настроенных VLAN, их идентификаторы (VLAN ID), имена и список портов, принадлежащих каждой виртуальной сети. Она позволяет быстро проверить конфигурацию VLAN и распределение интерфейсов по виртуальным сетям.

### 2. Охарактеризуйте VLAN Trunking Protocol (VTP). Приведите перечень команд с пояснениями для настройки и просмотра информации о VLAN.

**VLAN Trunking Protocol (VTP)** — это протокол второго уровня модели OSI, предназначенный для централизованного управления VLAN в сети, состоящей из нескольких коммутаторов. VTP позволяет автоматически распространять информацию о созданных VLAN между коммутаторами, объединёнными в один VTP-домен.

Основные режимы работы VTP:

- **Server** — коммутатор может создавать, изменять и удалять VLAN. Информация распространяется по сети.
- **Client** — коммутатор получает информацию о VLAN от сервера, но не может изменять её.
- **Transparent** — коммутатор не участвует в синхронизации VLAN, но пересылает VTP-сообщения дальше.

Основные команды настройки VTP:

vtp mode server  
— переводит коммутатор в режим VTP-сервера.

vtp mode client  
— переводит коммутатор в режим VTP-клиента.

vtp domain <имя_домена>  
— задаёт имя VTP-домена. Все коммутаторы должны иметь одинаковое имя домена.

vtp password <пароль>  
— устанавливает пароль для синхронизации базы VLAN.

vlan <номер>  
— создаёт новую VLAN.

name <имя_vlan>  
— задаёт имя VLAN.

Основные команды просмотра информации:

### 3. Охарактеризуйте Internet Control Message Protocol (ICMP). Опишите формат пакета ICMP.

**ICMP (Internet Control Message Protocol)** — это протокол сетевого уровня, используемый для диагностики сети и передачи служебных сообщений об ошибках. Он является частью стека TCP/IP и используется, например, утилитами **ping** и **traceroute**.

Основные функции ICMP:

- проверка доступности узлов сети;
- уведомление о сетевых ошибках;
- передача диагностической информации.

Формат ICMP-пакета включает следующие поля:

- **Type** — тип сообщения (например, Echo Request или Echo Reply);
- **Code** — дополнительная информация о типе сообщения;
- **Checksum** — контрольная сумма для проверки целостности данных;
- **Identifier** — идентификатор запроса;
- **Sequence Number** — номер последовательности пакета;
- **Data** — полезная нагрузка.

Сообщения ICMP инкапсулируются внутри IP-пакета и передаются по сети как обычные IP-данные.

### 4. Охарактеризуйте Address Resolution Protocol (ARP). Опишите формат пакета ARP.

**ARP (Address Resolution Protocol)** — это протокол канального уровня, предназначенный для определения MAC-адреса устройства по его IP-адресу внутри одной локальной сети.

Когда устройство хочет отправить пакет другому узлу, оно сначала проверяет ARP-таблицу. Если MAC-адрес неизвестен, устройство отправляет **ARP-запрос (ARP Request)** всем узлам сети. Устройство с соответствующим IP-адресом отвечает **ARP-ответом (ARP Reply)**, содержащим свой MAC-адрес.

Формат ARP-пакета включает следующие поля:

- **Hardware Type** — тип аппаратного интерфейса (например, Ethernet);
- **Protocol Type** — тип протокола верхнего уровня (IPv4);
- **Hardware Size** — длина аппаратного адреса (MAC);
- **Protocol Size** — длина IP-адреса;
- **Opcode** — тип операции (запрос или ответ);
- **Sender MAC Address** — MAC-адрес отправителя;
- **Sender IP Address** — IP-адрес отправителя;
- **Target MAC Address** — MAC-адрес получателя;
- **Target IP Address** — IP-адрес получателя.

### 5. Что такое MAC-адрес? Какова его структура?

**MAC-адрес (Media Access Control Address)** — это уникальный физический адрес сетевого интерфейса, используемый на канальном уровне модели OSI для идентификации устройств в локальной сети.

MAC-адрес состоит из **48 бит (6 байт)** и обычно записывается в шестнадцатеричном формате, например:

00:1A:2B:3C:4D:5E

Структура MAC-адреса:

- **Первые 24 бита (OUI — Organizationally Unique Identifier)**  
  идентификатор производителя сетевого оборудования.

- **Последние 24 бита**  
  уникальный номер интерфейса, назначаемый производителем.
