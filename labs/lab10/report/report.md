## Контрольные вопросы

### 1. Как задать действие правила для конкретного протокола?

Действие правила для конкретного протокола задаётся при создании записи в расширенном ACL с указанием нужного протокола.  

Например:
- `permit tcp` — разрешить трафик по протоколу TCP;
- `permit udp` — разрешить трафик по протоколу UDP;
- `permit icmp` — разрешить ICMP-сообщения.

Также можно использовать `deny` для запрета. Таким образом, протокол указывается непосредственно в команде ACL, что позволяет точно контролировать тип передаваемого трафика.

### 2. Как задать действие правила сразу для нескольких портов?

Для задания действия сразу для нескольких портов используется ключевое слово `range`, позволяющее указать диапазон портов.  

Например:
- `permit tcp any host 10.128.0.2 range 20 21` — разрешает доступ по портам 20 и 21 (FTP);
- также можно добавлять несколько отдельных правил для разных портов (например, отдельно для 80, 25, 110).

### 3. Как узнать номер правила в списке прав доступа?

Номер правила можно узнать с помощью команды:
- `show access-lists`

В выводе отображается список ACL с порядковыми номерами строк (sequence numbers), которые определяют порядок обработки правил. Эти номера используются для управления и редактирования списка.

### 4. Каким образом можно изменить порядок применения правил в списке контроля доступа?

Порядок применения правил можно изменить следующими способами:

- указать номер строки (sequence number) при добавлении правила, чтобы разместить его в нужной позиции списка;
- удалить правило и добавить его заново в нужное место;
- создать новый ACL с правильным порядком правил.

Так как правила ACL обрабатываются сверху вниз, изменение их порядка напрямую влияет на поведение фильтрации трафика.
