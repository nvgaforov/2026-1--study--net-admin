Ход выполнения

Исходная топология сети

Подключение ноутбука администратора

  • Ноутбук admin-donskaya-ngaforov-1
  • Подключён к порту FastEthernet0/24 коммутатора msk-donskaya-ngaforov-sw-4

Проверка параметров узла admin

  • Интерфейс FastEthernet0 находится в состоянии Up
  • Адрес 10.128.6.200/24 назначен корректно
  • Шлюз и DNS заданы правильно
  • Узел готов к административному управлению сетью

Настройка ACL для серверной сети

  • На маршрутизаторе создан ACL servers-out
  • Разрешён доступ:
    • ко web-серверу по HTTP
    • к mail-серверу по SMTP и POP3
    • к DNS-серверу по UDP/53
    • в сторону серверов по ICMP
  • Администратору дополнительно открыт доступ по FTP и Telnet

ACL для сети Other и сети управления

  • Создан ACL other-in
    • пользователям сети Other запрещён выход за пределы своей сети
    • для администратора оставлено разрешение
  • Создан ACL management-out
    • доступ к сети 10.128.1.0/24 разрешён только администраторам

Проверка созданных списков доступа

  • Выполнена команда show access-lists
  • Подтверждён порядок правил и их привязка к соответствующим интерфейсам

Проверка доступа к web-серверу

  • С пользовательского узла открыт адрес: http://www.donskaya.rudn.edu
  • Страница web-сервера загружается успешно
  • Это подтверждает правило: permit tcp any host 10.128.0.2 eq 80

Проверка FTP-доступа к web-серверу

  • Обычный пользователь:
    • команда ftp 10.128.0.2
    • соединение завершается по Timed out
  • Администратор:
    • выполняет вход по FTP
    • успешно проходит аутентификацию

FTP-доступ администратора

Проверка второго администратора и file-сервера

Проверка второго администратора и file-сервера
