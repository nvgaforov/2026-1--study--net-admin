## Контрольные вопросы

### 1. В чём состоит основной принцип работы NAT (что даёт наличие NAT в сети организации)?

**NAT (Network Address Translation)** — это механизм преобразования IP-адресов, при котором внутренние (частные) адреса локальной сети заменяются на внешние (публичные) при выходе в Интернет.

Основной принцип заключается в том, что маршрутизатор изменяет IP-адрес отправителя (а при необходимости и порт), сохраняя соответствие между внутренним и внешним соединением.

Использование NAT даёт следующие преимущества:

- экономия публичных IP-адресов;
- возможность выхода в Интернет для множества устройств через один или несколько внешних адресов;
- скрытие внутренней структуры сети (повышение безопасности);
- контроль доступа пользователей к внешним ресурсам.

### 2. В чём состоит принцип настройки NAT (на каком оборудовании и что нужно настроить для из локальной сети во внешнюю сеть через NAT)?

NAT настраивается на **маршрутизаторе**, который соединяет локальную сеть с внешней сетью (Интернетом).

Для настройки необходимо:

- определить внутренние (**inside**) и внешние (**outside**) интерфейсы;
- создать список доступа (ACL), который определяет, какие внутренние адреса будут транслироваться;
- при необходимости создать пул внешних IP-адресов;
- настроить правило трансляции, например:
  - динамический NAT;
  - PAT (NAT с перегрузкой);
  - статический NAT.

Для выхода из локальной сети во внешнюю обычно используется команда:

**ip nat inside source list <ACL> pool <POOL> overload**

Это позволяет нескольким внутренним устройствам использовать один или несколько внешних IP-адресов.

### 3. Можно ли применить Cisco IOS NAT к субинтерфейсам?

Да, NAT в Cisco IOS можно применять к **субинтерфейсам**.

Субинтерфейсы часто используются при работе с VLAN (например, схема «router-on-a-stick»). Для них можно задавать роли:

- **ip nat inside** — для внутренних сетей;
- **ip nat outside** — для внешнего интерфейса.

### 4. Что такое пулы IP NAT?

**Пул NAT** — это диапазон публичных IP-адресов, который используется для трансляции внутренних адресов во внешнюю сеть.

Пул задаётся на маршрутизаторе и может содержать один или несколько адресов. При динамическом NAT каждому соединению назначается свободный адрес из пула.

Пример:

**ip nat pool main-pool 198.51.100.2 198.51.100.14 netmask 255.255.255.240**

Пулы позволяют гибко управлять распределением внешних адресов между внутренними пользователями.

### 5. Что такое статические преобразования NAT?

**Статический NAT** — это постоянное соответствие между внутренним (частным) и внешним (публичным) IP-адресом или портом.

Такое преобразование настраивается вручную и не изменяется со временем. Оно используется для публикации внутренних серверов во внешнюю сеть.

Примеры:

- сопоставление IP-адресов (1:1);
- проброс портов (port forwarding), например:
  - внешний адрес и порт → внутренний сервер и порт.

Статический NAT позволяет обеспечить доступ извне к внутренним сервисам (WEB, FTP, почта), при этом контролируя, какие именно порты и ресурсы будут доступны.
