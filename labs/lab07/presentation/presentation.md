# Цель работы

## Цель

Получить навыки работы с физической рабочей областью Cisco Packet Tracer  
и изучить влияние физических параметров среды передачи на сеть.

# Ход выполнения

## Создание физической структуры

- Задан город **Moscow**  
- Созданы здания:
  - Donskaya  
  - Pavlovskaya  

![Город и здания](Screenshot_1.png){ width=70% }

## Размещение оборудования

- Добавлено серверное помещение (Wiring Closet)  
- Размещены коммутаторы и ПК  
- Сформирована топология сети  

![Размещение оборудования](Screenshot_3.png){ width=70% }

## Перемещение устройств

- Перемещены устройства в Pavlovskaya:
  - коммутатор  
  - два ПК  
- Использован инструмент Move  

![Перемещение устройств](Screenshot_4.png){ width=70% }

## Проверка соединения

- Выполнен ping между коммутаторами  
- Потерь пакетов нет  
- Соединение работает корректно  

![Ping успешен](Screenshot_6.png){ width=70% }

## Учёт длины кабеля

- Включена опция:
  - Enable Cable Length Effects  
- Устройства разнесены на ~1000 м  
- Используется медный кабель  

![Длинное соединение](Screenshot_7.png){ width=70% }

## Потеря связи

- Выполнен ping  
- Соединение не работает  
- Превышена допустимая длина кабеля  

![Нет связи](Screenshot_8.png){ width=70% }

## Добавление повторителей

- Добавлены 2 Repeater-PT  
- Назначены имена:
  - msk-donskaya-mc-1  
  - msk-pavlovskaya-mc-1  
- Настроены модули:
  - оптоволокно  
  - витая пара  

![Repeater настройка](Screenshot_9.png){ width=70% }

## Новая схема подключения

- Switch → витая пара → Repeater  
- Между репитерами → оптоволокно  
- Repeater → витая пара → Switch  

![Новая топология](Screenshot_11.png){ width=70% }

## Восстановление связи

- Выполнен повторный ping  
- Потерь пакетов нет  
- Соединение восстановлено  

![Ping успешен](Screenshot_12.png){ width=70% }

# Итог

## Вывод

- Изучены физические параметры сети  
- Выявлено ограничение длины медного кабеля  
- Применены повторители  
- Использовано оптоволокно  
- Обеспечена стабильная передача данных
